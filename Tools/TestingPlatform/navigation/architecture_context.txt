АРХІТЕКТУРНА СПЕЦИФІКАЦІЯ СИСТЕМИ НАВЕДЕННЯ ТА ОЦІНКИ ДИСТАНЦІЇ (EDGE AI)

ЗАГАЛЬНИЙ ОГЛЯД (HIGH-LEVEL DESIGN)
Система побудована за патерном Асинхронний подвійний контур (Dual-Loop Asynchronous Architecture) для гарантії стабільного FPS на слабкому залізі (Raspberry Pi).

Main Loop (Процес А): Синхронний, критичний до часу цикл. Виконує захоплення, детекцію (за потреби), математичний трекінг та розрахунок дистанції зі згладжуванням. Не містить жодних блокуючих викликів важких нейромереж, крім первинної ініціалізації.

Background Worker (Процес Б): Асинхронний ізольований процес (або потік). Приймає вирізані кадри (crops) по IPC, класифікує тип/ракурс дрона і повертає його фізичний розмір.

УПРАВЛІННЯ СТАНОМ (TARGET STATE)
Програма повинна мати глобальний (у межах Процесу А) об'єкт стану цілі — TargetState, який містить:

status: SEARCHING | TRACKING | LOST

bbox: (x, y, w, h) (поточні координати)

physical_size_mm: float (поточний очікуваний розмір цілі, ініціалізується дефолтним "середнім" значенням)

estimated_distance_m: float (відфільтрована дистанція)

ТОЧКА ВХОДУ: camera_tick_processor.py
Цей файл є ядром оркестрації. Метод tick(frame) викликається на кожному новому кадрі з камери. Він діє як State Machine.
Очікуваний час виконання: < 30ms.

Алгоритм виконання tick(frame):

Перевірка вхідних даних з Процесу Б (Non-blocking):
Перевірити IPC-канал (ZMQ). Якщо є нове повідомлення від класифікатора — оновити TargetState.physical_size_mm.

Обробка стану SEARCHING або LOST:
Викликати DetectionModule.detect(frame) (швидка YOLOv8n INT8).
Якщо ціль знайдено: оновити TargetState.bbox, ініціалізувати TrackingModule.init(frame, bbox), змінити статус на TRACKING.
Якщо не знайдено: повернути поточний TargetState і завершити tick.

Обробка стану TRACKING:
Викликати TrackingModule.update(frame).
Якщо трекер загубив ціль (confidence drop): змінити статус на LOST.
Якщо ціль успішно ведеться: оновити TargetState.bbox.

Розрахунок дистанції (Distance Estimation):
Передати поточний bbox та physical_size_mm у DistanceEstimator.calculate().
Оновити TargetState.estimated_distance_m.

Делегування (IPC Dispatch):
Кожні N кадрів вирізати crop (зображення цілі по bbox + padding) та відправити через неблокуючий виклик у ClassificationWorker (Процес Б).

Повернення результату:
Віддати оновлений TargetState системі керування приводами (PID).

ДЕКОМПОЗИЦІЯ МОДУЛІВ (DEPENDENCY MODULES)

4.1. detector.py (Fast Detector)

Відповідальність: Пошук дрона на повному кадрі для ініціалізації трекера.

Інтерфейс: detect(frame) -> bbox | None

Реалізація: Завантаження .tflite (або .onnx) моделі. Інференс оптимізований (ресайз вхідного кадру до 320x320).

4.2. tracker.py (Math Tracker)

Відповідальність: Пофреймове ведення цілі за допомогою класичного CV (OpenCV).

Інтерфейси: init(frame, bbox) -> None; update(frame) -> (success: bool, bbox).

Реалізація: Використання cv2.TrackerCSRT_create() або cv2.TrackerKCF_create().

4.3. distance_estimator.py (Math & Filtering)

Відповідальність: Обчислення дистанції до цілі, стійке до зміни ракурсу дрона.

Ключова логіка:
Обчислення піксельної діагоналі: D_px = sqrt(width^2 + height^2).
Базова монокулярна формула: Distance = (focal_length * physical_size_mm) / D_px.

Кінематичний фільтр: Зберігати попередні розрахунки та застосовувати Експоненційне ковзне середнє (EMA) або 1D Фільтр Калмана.

4.4. ipc_communicator.py (ZeroMQ Wrapper)

Відповідальність: Забезпечення неблокуючої передачі зображень та отримання результатів між процесами.

Специфікація: import zmq. Паттерн PUSH/PULL або REQ/REP з таймаутом NOBLOCK. Формат туди — numpy масив (crop), формат назад — JSON.

4.5. classification_worker.py (Повільний контур - Процес Б)

Відповідальність: Окрема програма. Безкінечний цикл, який слухає IPC.

Логіка: Очікує crop, проганяє через важчу мережу, маппить клас на фізичний розмір через словник, відправляє розмір назад. Якщо в черзі кілька кадрів — бере лише найсвіжіший.

СПЕЦИФІКАЦІЯ СЛОВНИКА РОЗМІРІВ (constants.py)
Словник має бути розгорнутим з урахуванням ракурсу:

DEFAULT_AVERAGE_DRONE: 350.0

MAVIC_3_FRONT: 347.0

MAVIC_3_SIDE: 280.0

FPV_7_INCH_FRONT: 320.0

FPV_7_INCH_SIDE: 250.0

Інструкція для AI: Будуй архітектуру з жорсткою ізоляцією залежностей (Dependency Injection). camera_tick_processor.py не повинен знати деталі реалізації YOLO або ZMQ, він має оперувати лише абстрактними інтерфейсами.
